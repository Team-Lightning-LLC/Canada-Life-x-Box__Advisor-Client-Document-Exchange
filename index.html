Secure Document Exchange — POC

Three workflows. One platform.

Every use case follows the same pattern — someone needs a document, someone uploads it, someone reviews it, and the status tracks itself. Box is the engine. These are the workflows.

1
Claims — Additional Documentation
A claims examiner pauses a claim needing additional documents. The member uploads securely, the submission ties back to the original case, and Appian workflow resumes automatically.

Advisor — Client Document Exchange
Advisor collects docs from a client. Ops reviews, rejects with reasons, client re-uploads the specific document. DocuSign fires when needed. Status visible everywhere.

3
Third-Party Secure Upload
System sends a request to an external party — a doctor, lab, or legal firm. They upload securely without credentials. Access is scoped, time-limited, and auto-expires.
System External Party Reviewer
Tiered auth: File Request → Collab → Scoped Token
Auto-expiring access with audit trail

One architecture serves all three. Box folders are the data model. Metadata templates are the schema. Relay is the workflow engine. UI Elements are the frontend. The variables are who initiates, who uploads, and which system gets notified. The pattern is identical.
